## ASVS L1 Security Posture

Auditability, integrity of security evidence, bounded resource consumption, safe failure and prevention of misleading success are blocking ASVS L1 controls. Any unresolved HIGH threat blocks this plan.

## Trust Boundaries

| Boundary | Untrusted crossing | Control |
|---|---|---|
| Manifest → adapter registry | domain/ID/status/bounds | closed schema, logical IDs, digest frozen before window |
| Surface → snapshot | synthetic and exact real filesystem/service state | bounded allowlisted adapters, proof freshness and opaque per-run HMAC |
| Snapshot pair → verdict | missing/different/malformed evidence | strict four-state evaluator |
| Verdict → report | claim/evidence fields | passed-only report and exact claim allowlist |

## STRIDE Threat Register

| Threat ID | STRIDE | Severity | Component | Disposition | Concrete mitigation and blocking proof |
|---|---|---:|---|---|---|
| T-01 | Spoofing / Tampering | HIGH | manifest/evidence identity | mitigate | Exact manifest/suite/tier/window digests and token pairing; substitution tests block completion. |
