## Phase Goal

**As a** Mac 配置仓库操作者，**I want to** 校验严格区分且有 lineage 的恢复 artifacts，并在隔离 fixture 中运行端到端安全测试，**so that** 我能证明本次运行覆盖的真实 Mac 表面没有发生未经授权的变化。

<objective>
在所有 CLI renderer、stderr diagnostic 与 artifact store 之前建立同一个 fail-closed privacy boundary，并为 synthetic fake adapter 增加有界、内存内、无 shell 的 raw capture。

Purpose: 落实 SAFE-02、SAFE-03 与 D-05～D-08，证明隐私违规不会先落盘/显示再依赖 masking 补救。
Output: logical-reference grammar、registered normalization、safe error envelope、pre-output gate、bounded capture、canary/raw fixtures 与端到端泄漏负测试。
</objective>

<threat_model>
## ASVS L1 Security Posture

Safe output encoding, sensitive-data protection, strict input validation, process execution allowlisting and bounded resource use are blocking ASVS L1 controls. Any unresolved HIGH threat blocks this plan.

## Trust Boundaries

| Boundary | Untrusted crossing | Control |
|---|---|---|
| Artifact/CLI candidate → output | logical refs and arbitrary fields | one pre-output gate; closed normalizations |
| Privacy failure → stderr | diagnostic metadata | exact safe error schema, no derived fingerprint |
| Fake process → adapter | stdout/stderr bytes | fixed registry, timeout/size caps, memory-only parse/discard |
| Logical resolver → artifact | physical path/identity | process-local mapping; logical namespace only |

## STRIDE Threat Register

| Threat ID | STRIDE | Severity | Component | Disposition | Concrete mitigation and blocking proof |
|---|---|---:|---|---|---|
| T-01 | Spoofing / Tampering | HIGH | artifact input | mitigate | Reuse strict kind/digest/lineage validation before privacy-approved output. |
| T-02 | Information Disclosure | HIGH | store/stdout/stderr/raw capture | mitigate | D-05..D-08 shared gate, logical refs, safe error envelope and canary tests across all sinks. |
| T-03 | Tampering / Elevation of Privilege | HIGH | subprocess execution | mitigate | Registry-owned executable and argv only, no shell/arbitrary command, normalized typed output only. |
| T-04 | Tampering / Elevation of Privilege | HIGH | resolver/domain/fixture paths | mitigate | Six-namespace grammar, closed surface-domain compatibility and canonical containment reject wrong-domain refs, traversal, absolute refs and resolver/symlink escape before every sink. |
| T-05 | Elevation of Privilege / Information Disclosure | HIGH | process/network environment | mitigate | Runner allowlist, no proxy/credentials/network; capture cannot escalate tier or retain raw state. |
| T-06 | Repudiation / Denial of Service | HIGH | capture failure/evidence | mitigate | Fixed timeout/byte caps, deterministic `unknown`/privacy error, non-zero result and no retry-to-pass. |

## Security Gate

Any canary or derived fingerprint in stdout, stderr, store, test failure text or retained fixture is a HIGH unresolved disclosure and blocks commit. Any arbitrary command, unbounded capture or path escape likewise blocks completion.
