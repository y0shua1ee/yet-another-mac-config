<threat_model>
## ASVS L1 Security Posture

Input validation, canonical serialization, integrity checks, safe error handling and deny-by-default state transitions are mandatory ASVS L1 controls. Any unresolved HIGH threat blocks execution.

## Trust Boundaries

| Boundary | Untrusted crossing | Control |
|---|---|---|
| JSON bytes → envelope | kind/version/fields/numbers | closed decoder, duplicate-key and trailing-token rejection |
| Envelope → canonical digest | caller-provided content | restricted canonical form and recomputed SHA-256 |
| Artifact graph → evidence | upstream digest references | schema-defined exact edges and mode-specific rules |
| Store path → content | filename/key claims | recompute on read/write; immutable plan key |

## STRIDE Threat Register

| Threat ID | STRIDE | Severity | Component | Disposition | Concrete mitigation and blocking proof |
|---|---|---:|---|---|---|
| T-01 | Spoofing / Tampering | HIGH | kind/schema/lineage/lifecycle | mitigate | Six closed payload decoders, closed class-kind-retention table, duplicate-key rejection, canonical digest and exact graph; cross-kind, wrong-class, unsupported-retention, stale and latest negative suites block completion. |
| T-02 | Information Disclosure | HIGH | validation errors | mitigate | Stable code and logical pointer only; invalid raw bytes are never stored or echoed. Full reusable privacy envelope arrives in Plan 01-03. |
| T-03 | Tampering / Elevation of Privilege | HIGH | generated-plan payload | mitigate | Data-only typed payload, no raw command fields, immutable plan keys, no apply route. |
| T-04 | Tampering / Elevation of Privilege | HIGH | artifact store filesystem | mitigate | Reuse Plan 01-01 external-root containment; store-key mismatch and overwrite tests are blocking. |
| T-05 | Elevation of Privilege / Information Disclosure | HIGH | validation runner | mitigate | Offline local Go environment only; no network, manager or live escalation path. |
| T-06 | Repudiation / Denial of Service | HIGH | evidence/report graph and retention | mitigate | Report lists exact validated evidence only; 24-hour snapshot expiry, plan terminal states and transitive pins are enforced on every operation; premature/pinned/evidence deletion and missing/freshness failures are non-zero. |

## Security Gate

All T-01..T-06 mitigations must have passing negative tests. Any partial artifact, quarantine write, key mismatch, latest selection or lineage ambiguity blocks both task commits.
